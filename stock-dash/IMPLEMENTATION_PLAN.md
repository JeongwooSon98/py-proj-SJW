# 🛠 Stock Dashboard 기능 확장 구현 계획서

전문 트레이더 관점에서 도출한 개선안을 **단계적으로** 구현하기 위한 작업 계획서.
각 Phase는 독립적으로 머지/검증 가능하도록 쪼갰으며, 위에서부터 순서대로 진행한다.

> **공통 규칙 (CLAUDE.md 준수)**
> - 점수·임계값은 `computeSignal`, 지표 파라미터는 `computeIndicators`에 모은다.
> - 점수 체계를 바꾸면 **반드시 [INDICATORS.md](INDICATORS.md)** 와 UI 도움말(`*_INFO`)을 같이 고친다.
> - 가중치 위계 유지: **타이밍(±3) > 위치(±2) > 추세·확인 보조(±1)**.
> - 각 Phase 종료 시 `npm run build`로 검증한다.
> - 응답·주석·문서는 한국어.
> - **배지 클릭 동작은 신호 추가 시점에 함께 구현한다** (몰아서 X). 신호가 특정 봉(날짜)에 묶이면 `chart`/`date`를 줘 **차트 이동·하이라이트(↗)**, 그렇지 않으면(주봉·시장 기준 등) `explain` 문구를 줘 **설명 팝오버(ⓘ)** 로 "왜 차트로 못 가는지 + 신호 의미"를 보여준다. 모든 배지는 클릭 가능해야 한다.

## 🔴 Phase 1 — ATR 기반 리스크 관리 (손절 / 목표가 / 손익비)

**목표**: "언제 살까"에 더해 **"어디서 자르고 어디서 익절할까"** 를 제시한다. 점수 체계는 건드리지 않으므로 리스크가 낮아 1순위.

### 변경 사항
- [x] `computeIndicators`: **ATR(14)** 추가 (True Range의 Wilder 평활).
  - `TR = max(high-low, |high-prevClose|, |low-prevClose|)`
  - 각 행에 `out[i].atr` 저장.
- [x] `computeSignal`: 마지막 봉 기준 리스크 레벨 계산해 반환값에 추가.
  - `atr`, `atrPct`(= atr/close×100)
  - 매수 우위(score ≥ 0): `stop = close − 2×ATR`, `target = close + 3×ATR`
  - 매도 우위(score < 0): `stop = close + 2×ATR`, `target = close − 3×ATR`
  - `rr`(손익비) = |target−close| / |close−stop| (배수 2/3 기준이면 1.5 고정이지만, 향후 지지/저항 기반 확장 대비해 계산식으로 둠)
- [x] `SignalCard` UI: 메타 줄 아래에 **손절 / 1차 목표 / 손익비(R:R)** 한 줄 추가. 변동성(ATR%)도 작게 표기.
- [x] **INDICATORS.md**: "리스크 관리 (ATR)" 섹션 신설 — ATR 정의, 손절·목표 배수(2×/3×) 근거, R:R 해석. (⑤로 수록)

### 정합성 체크
- 점수/등급 변화 **없음** (정보 표시만 추가).

**검증**: 빌드 통과 + 추천 카드에 손절/목표/손익비 노출.

## 🟠 Phase 2 — 멀티 타임프레임 필터 + 상대강도(RS)

**목표**: 일봉 신호의 신뢰도를 주봉 추세로 거르고, 시장 대비 강한 종목을 가산한다. **점수 체계가 바뀌므로** 만점·등급 재조정 필수.

### 변경 사항
- [x] `loadAllSignals`: 종목별로 일봉(1y)에 더해 **주봉(2y, 1wk)** 시계열도 받아 주봉 추세 컨텍스트 계산.
  - 주봉 추세 = `주봉 MA20 > 주봉 MA60` → 상승 / 반대 → 하락 / 그 외 중립.
- [x] **벤치마크 상대강도(RS)**: 미국 종목은 `SPY`, 한국 종목(`*.KS`)은 `^KS11` 대비 최근 N일(예: 60일) 상대 수익률. 벤치마크 시계열은 한 번만 받아 캐시 후 재사용.
- [x] `computeSignal` 시그니처 확장: `computeSignal(rows, { weeklyTrend, rs })`. (이후 `rsShort/rsLong/regimeRisk/regimeBull`로 확장)
  - **신규 신호**:
    - `주봉 추세 일치(+1 / −1)` — 일봉 매수신호인데 주봉도 상승이면 +1 (반대 −1). 추세 필터(보조) 등급.
    - `시장 대비 강세(+1) / 약세(−1)` — RS 상위면 +1.
  - 주봉이 매수 방향과 **반대**면 일봉 매수 신호를 감점하는 패널티 룰도 고려(설계 시 결정). → 별도 패널티 대신 **주봉 하락추세 −1 독립 신호**로 실현(점수 상쇄 효과 동일).
- [x] **만점 재계산 & `signalGrade` 임계값 재조정**: 신호 2종(±각1) 추가로 매수 만점 +12→+14, 매도 −11→−13. (Phase 2 당시값. 최종 Phase 3에서 +17/−16) 등급컷은 비율 맞춰 ±7/±3으로 정착.
- [x] **INDICATORS.md**: 멀티 타임프레임·RS 신호를 점수표에 추가, 만점·등급표 갱신. (⑥로 수록)

### 정합성 체크
- 신호 추가 → 만점·등급·노출 임계값 3곳 동시 갱신 확인.

**검증**: 빌드 통과 + 카드에 "주봉↑", "시장대비 강세" 배지 노출 + 등급 분포가 합리적인지 눈으로 확인.

## 🟢 Phase 2.5 — 상대강도(RS) 정교화 (Phase 2 보완)

**배경**: 현재 RS는 `(현재 종가 − 60일 전 종가)`의 **두 끝점만** 비교한다. 이 방식은 **경로를 무시**해서, 앞 두 달 급등 후 최근 1~2주 추세가 꺾여 급락한 종목도 누적 초과수익이 (+)면 여전히 "시장 대비 강세"로 오판한다. (endpoint sensitivity·최근 모멘텀 반전 미반영). 오닐식 RS도 단일 구간이 아니라 최근 구간에 가중을 둔다 → "장기 우위 + 최근에도 안 무너짐"을 함께 봐야 한다.

### 변경 사항
- [x] **A안(권장·단순)**: 단기(20일) + 중기(60일) RS를 **둘 다** 계산. ← 채택
  - 둘 다 ≥ +3%p → **강세(+1)** / 둘 다 ≤ −3%p → **약세(−1)** / 엇갈리면 **중립(신호 미부여)**.
  - 최근 급락 시 단기 RS가 음전 → 강세에서 자동 탈락(사용자 지적 해소).
- [ ] (대안 B안) 가중 RS `0.4·R20 + 0.35·R60 + 0.25·R120` — 더 부드럽지만 복잡. **추후 검토(미채택)**.
- [x] `periodReturn`/`loadAllSignals`에서 20일 벤치마크 수익률도 함께 산출(벤치 호출 재사용).
- [x] RS 배지 `explain` 문구를 "단기·중기 동시 강세" 기준으로 갱신.
- [x] **INDICATORS.md**: ⑥ RS 설명을 단기+중기 동시 조건으로 수정. (만점·등급은 **불변** — 신호 개수·배점 그대로)

### 정합성 체크
- 점수 배점/만점/등급 **변화 없음**(RS 판정 조건만 정교화). 배지 클릭은 기존 explain 팝오버(ⓘ) 유지.

**검증**: 빌드 통과 + 최근 급락 종목이 더 이상 "시장 대비 강세"로 뜨지 않는지 확인.

## 🟡 Phase 3 — 시장 레짐(VIX·ADX) + 다이버전스

**목표**: 평균회귀 신호와 추세추종 신호가 시장 상태에 맞게 작동하도록 맥락을 추가. **(검증에서 실증된 "폭락장인데 다수가 매수 관심" 문제를 우선 보정)**

### 변경 사항
- [x] `computeIndicators`: **ADX(14)** 추가 (3-B 완료) → 추세장(ADX≥25)/횡보장(<20) 판별. +DM/−DM→+DI/−DI→DX→ADX(Wilder), TR은 ATR `trArr` 재사용.
- [x] 대시보드 상단에 **시장 레짐 바** (3-C 완료): `MarketRegimeWidget` — VIX(^VIX) 위험선호/회피 + S&P500(^GSPC) ADX 추세장/횡보장 한 줄 요약. `useMarketRegime` 훅(점수 무관·표시용).
- [x] **지수 레짐 보정** (3-A 완료): 벤치마크가 **하락추세(종가<MA60 AND MA20<MA60) 또는 급락(10일 ≤ −7%)** 이면 평균회귀 매수신호(RSI 과매도·볼린저 하단)를 **+2→+1 가중 축소**. `computeBenchRegime`로 판정 후 `computeSignal(ctx.regimeRisk)`에 전달. 만점·등급 불변(레짐 양호 시 점수 그대로). "확실한 위험만 좁게 발동"(약간 공격적 성향) + "무효화 아닌 절반"으로 V자 반등·추세 동반 반등은 살리고 평균회귀 단독 칼받이만 노출 임계 아래로 떨어뜨림.
- [x] **다이버전스 감지** (3-D 완료): 가격 저점↓ vs RSI/MACD 저점↑(강세 +2) / 반대(약세 −2). 좌우 3봉 스윙 피벗 직전 2개 비교(`computeIndicators`에서 `rsiDiv`/`macdDiv` 마커, stochCross와 동일 패턴), `computeSignal`이 최근 12봉 스캔해 단일 ±2 채점. RSI·MACD 차트에 ◇ 마커, 배지 `chart`/`date`로 ↗ 이동. 만점 +14→+16 / −13→−15.
- [x] **레짐 가중** (완료): 단일 ±1 "레짐 정합" 확인 보너스. 종목 ADX≥25(추세장)+추세추종 신호 / ADX<20(횡보장)+평균회귀 신호 정합 시 종합 점수 부호 방향으로 ±1. 개별 신호 점수·위계 불변, 천장만 +1(매수 +16→+17 / 매도 −15→−16). ⑦(벤치 추세)과 독립. 배지 ⓘ 팝오버. 등급컷 ±7/±3·노출 ±3 유지.
- [x] **INDICATORS.md** (완료): ⑨ 다이버전스·⑩ 레짐 정합 보너스 섹션 추가, 점수표(#9·#10)·만점(+17/−16) 갱신. SIGNAL_TIPS·차트 마커·CLAUDE.md 만점 수치 동기화.

### 정합성 체크
- 신호/가중 변경 → 만점·등급·문서 동시 갱신. 신규 배지(다이버전스=↗ / 레짐은 카드 상단 요약) 클릭 동작 함께 구현.

**검증**: 빌드 통과 + 레짐 요약·다이버전스 마커 노출 + 폭락장에서 매수 관심 종목이 합리적으로 줄어드는지 확인.

## 🔵 Phase 4 — 간이 백테스트 (신호 신뢰성 검증)

**목표**: 현재 점수 체계가 **실제로 맞는지** 데이터로 검증. 앞으로의 가중치 조정 근거를 만든다.

### 변경 사항
- [x] `backtestSeries(ind)`: 과거 각 봉 `i`에서 **`computeSignal(ind.slice(0,i+1))`을 재호출**해 그 시점 신호를 재현(점수·임계값 로직 중복 없이 재사용). 직전 봉엔 없다 새로 켜진 신호 라벨을 진입으로 잡아 **5·10·20봉 후 종가 수익률** 집계.
  - 개별 신호 + 종합 점수 구간별(±7/±3 경계) 평균 수익률·승률·표본 수 산출(`aggregateBacktest`).
  - 진입=신호 첫 발생 봉 종가. 매수(▲)=상승, 매도(▼)=하락을 적중으로 부호 정렬(승률=적중 비율).
- [x] **"신호 검증" 접이식 섹션** (`BacktestPanel`, 추천 패널 아래) 추가 — 개별 신호 표 + 구간 표, 각 행에 **표본수(n) 노출**(과최적화 경계).
- [x] 무거운 연산이라 종목 합산 결과를 **별도 캐시**(`backtestCache`, 30분) + **lazy 실행**(펼칠 때 1회). `useBacktest` 훅.
- [x] 표본 범위: **3년 일봉 · 15종목 전체 합산**(1y는 표본 부족).
- [x] **INDICATORS.md**에 "2-b. 신호 검증(백테스트)" 섹션 추가 — 방법론(진입·보유기간·표본·방향)·구현 원리·한계(컨텍스트 신호 제외·다이버전스 look-ahead·거래비용 미반영) 문서화.

### 정합성 체크
- 점수 체계 자체는 변경 없음(검증 도구). 만점(+17/−16)·등급컷·노출 임계 **불변**. 결과가 나쁜 신호는 별도 합의 후 가중치 재조정 검토.

**검증**: 빌드 통과 ✅ + 헤드리스 런타임 표 노출/표본 수·수익률 상식성 확인.

> **설계 결정(사용자 합의)**: ①검증 대상 = 개별 신호 + 종합 점수 구간 둘 다 / ②진입 = 신호 첫 발생 봉 종가 / ③청산 = 고정 5·10·20봉 후 종가 / ④표본 = 3년 일봉·종목 전체 합산 / ⑤UI = 추천 패널 아래 별도 접이식 섹션(n 필수 노출).

## ⚪ Phase 5 — 펀더멘털 / 이벤트 (FMP 연동, 선택)

**목표**: 기술적 신호에 펀더멘털·이벤트 맥락을 결합. **데이터 소스 개편**과 함께 진행.

### 데이터 타당성 실측 결과 (FMP MCP, 착수 전 조사 — 설계 근거)
- **데이터 소스 전략 = 이중화(a안 채택)**: 검증된 시세·차트·추천 신호는 **Yahoo 유지**, 펀더멘털만 **FMP** 추가. 전면 이관(b)은 위험 대비 실익 없음.
- **FMP 런타임 접근 = 빌드타임 정적 JSON(채택)**: MCP는 *에이전트*만 호출 가능 → 브라우저 앱은 불가. 런타임 REST는 API 키 노출·CORS·무료 250/day 한계. 그래서 `src/data/fundamentals.json` 으로 **빌드타임에 구워** 정적 import(키는 스크립트에만). `scripts/fetch-fundamentals.mjs`(FMP_API_KEY)로 재생성.
- **커버리지 (하위 플랜 실측)**: ✅ 미국 개별주 밸류에이션(PER/PBR/PEG/배당)·애널리스트 목표가 / ❌ 어닝 캘린더(전 종목 플랜 차단) / ❌ 한국주(지역 차단) / ❌ AVGO(종목 차단) / — ETF(개념상 해당 없음). → 표시 대상 = **NVDA·TSLA·AAPL·MSFT·GOOGL·AMZN 6종**.

### 백테스트 기반 1차 보정 — ⑦ 레짐 대칭화 (Phase 4 후속, 점수 체계 첫 데이터 근거 조정)

- **발견**: Phase 4 간이 백테스트에서 거의 모든 **매도 신호의 N봉 후 승률이 30%대**로 일관되게 낮았다(볼린저 상단 34%·스토캐스틱 과매수 31%·RSI 과매수 33% 등, 20일 기준). 전문가 진단 결과 이는 ①3년 강세장 + 강세 성장주 편중 표본 ②시장의 우상향 드리프트(매수=순방향/매도=역방향)라는 **구조적 비대칭**이 대부분으로, 개별 매도 배점을 승률에 맞추는 미세조정은 **과최적화**로 판단해 배제.
- **진짜 결함 = 레짐 보정의 비대칭**: ⑦이 "지수 위험 → 매수 평균회귀 축소"만 있고 거울상인 "지수 강세 → 매도 평균회귀 축소"가 빠져, 강세장에서 **단일 과매수만으로 '매도 주의'가 과다 노출**되고 있었다.
- **보정(처방 A, 사용자 합의)**: `computeBenchRegime`에 **강세(bull)** 판정 추가(상승추세 `종가>MA60 AND MA20>MA60` 또는 급등 `10일≥+7%`, 급락의 대칭). `computeSignal`이 `ctx.regimeBull` 시 **RSI 과매수·볼린저 상단 −2→−1** 축소(매수쪽 −2→−1 칼받이 억제의 완전 대칭). 추세형 매도(역배열·MACD 데드·약세 다이버전스)는 **건드리지 않음**(진짜 추세 전환 보존).
- **불변**: 만점(+17/−16)·등급컷(±7/±3)·노출 임계(±3) 그대로(레짐 중립 시 점수 불변). INDICATORS.md ⑦ 대칭 표·점수표 매도 주석·SIGNAL_TIPS 2개 라벨 동기화.
- **보류(표본 부족)**: `MACD 데드크로스(0선 아래)` −3 재검토(20일 −7.8%·n94, 0선 아래=강세장 바닥권 반등 영역일 수 있음)와 약세 다이버전스 단기성은 **약세장 포함 표본 보강(C) 후** 재판단.

### 배지 UX 개선 (Phase 2.5 진행 중 사용자 피드백, 점수·만점·등급 불변)

- **explain 문구 간결화**: RS·주봉 추세 설명을 "핵심(신호 의미) → 주의(±1 보조신호일 뿐, 매수/매도 판단은 종합 점수·등급으로)" 2문단으로 압축. 차트 이동 관련 중복 문장 제거(팝오버 하단 고정 힌트가 처리).
- **주봉 추세 배지 → 차트 이동(↗)**: 클릭 시 차트를 **주봉(3년)으로 전환**해 MA20·MA60 배열을 보여줌. `reason.interval="1wk"` + `highlight.interval`로 `ChartPanel`이 봉/기간 전환. 추세 '상태'라 세로선 대신 가격 차트 스크롤·강조.
- **전 배지 hover 툴팁**: 마우스를 올리면 그 신호가 왜 매수/매도 신호인지 짧은 설명(`SIGNAL_TIPS` 맵, 라벨로 조회) + 클릭 동작 안내를 표시.

### 데이터 신선도 보정 (Phase 3 검증 중 발견·수정)

- **문제**: 일부 종목(특히 `*.KS`)은 Yahoo 일봉 시계열이 며칠 지연(예: 삼성전자 시계열 마지막 06-26 종가 339,500 vs 실시간 현재가 323,000, 5% 괴리)되어, MA·RSI·볼린저·레짐이 **묵은 종가로 계산**됨. (헤드리스 디버그로 실측 확인)
- **수정**: `fetchSeries`(일봉)에서 **마지막 봉 종가를 `meta.regularMarketPrice`로 갱신**(high/low 정합 유지). 새 봉 추가 방식은 미국 거래일↔KST 타임존 경계에서 중복 봉을 만들어 폐기하고, 봉 수를 늘리지 않는 갱신 방식 채택.
- **재검토 결과**: 레짐 판정/보정 로직 자체는 정상이었음(미국·한국 모두 10일·추세 기준 실제 위험 국면 아니어서 보정 미발동이 맞음). 화면 지수카드의 큰 −% 는 *1일* 등락률이라 레짐(10일·추세)과 별개.
