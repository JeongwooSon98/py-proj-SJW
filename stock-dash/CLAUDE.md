# CLAUDE.md

Vite + React 주식 보조지표 대시보드. 관심종목을 10개 신호로 채점해 **여러 지표의 방향 합의도를 점수화해 보여준다(예측력 미검증).** (일봉·1년·투자 참고용)

- 핵심 로직: [src/StockDashboard.jsx](src/StockDashboard.jsx) — 지표 계산 `computeIndicators`, 점수화 `computeSignal`
- 지표·점수 문서: [INDICATORS.md](INDICATORS.md)

## 🔴 이 앱이 주장할 수 있는 것과 없는 것

15년 일봉(하락장 포함)·양 레짐·거래비용 차감 검증 결과, **종합 점수(+3↑)의 초과수익은 +0.1%p** — 아무 날에나 산 것과 **구분되지 않는다**(→ [INDICATORS.md](INDICATORS.md) 2-b, [AUDIT_FIX_PLAN.md](AUDIT_FIX_PLAN.md)). 개별 신호도 대부분 ±0.5%p 이내(노이즈 범위)다.

- ✅ **주장 가능**: "지금 여러 지표가 같은 방향을 가리킨다"는 **사실 요약**. 이 앱의 가치는 여기 있고, **알파가 없어도 성립한다.**
- ❌ **주장 불가**: "이 종목이 오른다" / "지금 사면 된다". **예측·처방 문구를 UI·문서에 다시 넣지 말 것** — "추천 매수", "적극 매수", "오늘 매수 적합", "진입 무난", "비중 축소 유효" 등은 [REFRAME_PLAN.md](REFRAME_PLAN.md) R1~R4에서 **의도적으로 제거**했다.
- ⚠️ **"초과수익이 0이니 그 신호를 빼자"는 잘못된 추론이다.** '우위 없음'은 '해롭다'가 아니다. 신호 제거·가중치 변경은 **성과 근거가 아니라 설계 근거**(이중계상·문서-코드 불일치)로만 정당화된다.

## 점수 체계 유지 방안

- **Confluence 우선**: 단일 지표는 속임수가 많다. 여러 신호가 겹칠수록 총점이 커지는 합산 구조를 유지한다.
- **가중치 위계**: 타이밍(±3) > 위치(±2) > 추세·확인 보조(±1). 새 신호 추가 시 이 위계를 깨지 말 것.
- **노이즈 필터 유지**: 스토캐스틱은 Slow(14,3,3) + 과매도/과매수 강크로스만, MACD는 0선 필터로 신뢰도 차등.
- 임계값·가중치 변경은 백테스트 근거 없이는 보수적으로. **사후에 새 합격 기준을 만들어 붙이지 말 것(p-hacking).**

## ⚠️ 필수 규칙: 코드 ↔ 문서 동기화

지표 계산·점수·임계값·등급 기준을 코드에서 바꾸면 **반드시 같은 변경을 [INDICATORS.md](INDICATORS.md)에 반영**한다. 차트 라벨·`STOCH_INFO` 등 UI 도움말 문구도 함께 맞춘다. 코드만 고치고 문서를 두지 않는 것을 금지한다.

## 코드 수정 주의사항

- 점수·임계값은 전부 `computeSignal`, 지표 파라미터는 `computeIndicators`에 모아둔다. 흩뜨리지 말 것.
- 신호 추가 시 **만점(현재 매수 +17 / 매도 −16)** 과 등급 기준(`signalGrade`)·노출 임계값·`btScoreBucket` 경계와의 정합성을 확인한다.
- **등급 라벨**(`signalGrade`·`btScoreBucket`)은 "매수신호 강함/있음" 처럼 **신호가 모인 정도**만 말한다. 행동 권유("적극 매수")로 되돌리지 말 것.
- 수정 후 `npm run build`로 검증한다. **점수 로직을 건드리지 않았다면 `npm run backtest` 출력이 변경 전과 완전히 동일해야 한다** — 이게 무변경의 증거다.
- 사용자 응답·주석·문서는 한국어로 작성한다.
