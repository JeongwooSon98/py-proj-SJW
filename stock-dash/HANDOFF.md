### 방법론 교훈 (반복 금지)

1. **부분문자열 매칭으로 신호 방향을 판정하지 말 것.** `과매도`가 `/매도/`에 걸린다.
   명시적 집합(`Set`)이나 `dir` 필드를 쓴다. 앱 본체(`aggregateBacktest`)는 `dir` 필드로
   판정하므로 안전하다 — **일회성 스크립트에서만 재발한다.**
2. **일회성 스크립트의 결론을 앱 엔진(`npm run backtest`)으로 교차검증하기 전엔
   문서에 확정 사실로 쓰지 말 것.**
3. **초과수익 ±0.5%p는 이 표본에서 노이즈와 구분되지 않는다.** 겹치는 보유기간 + 종목 간
   상관 때문에 실질 독립 표본이 n보다 훨씬 적다. 유의성은 **날짜 블록 부트스트랩**
   (블록 25일 ≥ 최장 보유 20일)으로만 판단한다. 단순 t-검정 금지.
4. **점수 로직 무변경을 주장하려면 `npm run backtest` 출력 동일성으로 증명할 것.**
   REFRAME에서 이 방법으로 증명했다(라벨만 바꾼 2단계 검증 포함).

### 측정 스크립트 재현법

Node에서 Yahoo를 **직접** 호출한다 (corsproxy는 Node에서 실패):
```js
const r = await fetch(
  `https://query1.finance.yahoo.com/v8/finance/chart/${sym}?range=15y&interval=1d`,
  { headers: { "User-Agent": "Mozilla/5.0" } }
);
```
⚠️ `range=max`는 **월봉**이 온다. 장기 일봉은 `15y`·`20y`처럼 명시적 기간을 쓸 것.

일회성 분석 스크립트는 **프로젝트 안**(`scripts/.xxx-tmp.mjs`)에서 실행해야 한다 —
`esbuild`(vite 전이 의존성)를 Node가 프로젝트 `node_modules`에서 찾기 때문이다.
**끝나면 반드시 지울 것**(커밋 금지).
